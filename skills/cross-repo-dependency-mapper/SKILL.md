# Cross-Repo Dependency Mapper

This skill focuses on 'System Integrity', preventing conflicts between different repositories and clusters.

## EVOLUTION_LOGIC
1. Maintain a live 'Dependency Graph' of all skills across the 12+ GitHub repos.
2. Before creating a new skill, check if an existing skill already covers >50% of the logic.
3. If a conflict is found, merge the instructions instead of creating a duplicate.

## Examples
- "Check if our new 'Arbitrage' skill conflicts with the existing 'Sentinel' logic."
